# 프로젝트: 소설 집필 AI 에이전트 (Novel Writing Agent)

## 1. 프로젝트 개요
**소설 집필 AI 에이전트**는 장편 소설 작가들이 설정 충돌 없이 일관성 있는 스토리를 작성할 수 있도록 돕는 풀스택 웹 애플리케이션입니다. 단순한 텍스트 생성을 넘어, **RAG(검색 증강 생성) 기반의 컨텍스트 관리 시스템**을 구축하여 과거 에피소드, 등장인물(Story Bible), 챕터별 요약 정보를 AI가 종합적으로 인지한 상태에서 초안을 작성하고 교정할 수 있도록 지원합니다.

## 2. 주요 서비스 및 기능
*   **스토리 및 에피소드 관리:** 소설의 기본 메타데이터(장르, 시놉시스, 작풍)를 설정하고 챕터별 에피소드 작성 및 타임라인 관리.
*   **컨텍스트 인식 AI 초안 작성 (Context-Aware Drafting):** 설정 사전(Story Bible), 직전 에피소드 요약, 최근 에피소드의 슬라이딩 윈도우(Sliding Window)를 조합하여 일관성 있는 다음 챕터 초안 생성.
*   **스토리 바이블(설정집) 자동 추출 및 관리:** 에피소드 본문에서 AI를 활용해 인물, 장소, 아이템, 사건 등의 메타데이터를 JSON 형태로 자동 추출하고 DB에 저장 및 적용.
*   **RAG 기반 과거 문맥 검색:** PostgreSQL `pgvector`를 활용하여 에피소드 청크(Chunk)의 임베딩 벡터를 저장하고, 집필 중 유사한 과거 텍스트나 설정을 벡터 기반으로 검색하여 가져옴.
*   **AI 교정 도구 (일관성 검증 및 문체 변환):** 작성된 초안이 기존 설정과 충돌하지 않는지 검토하고, 특정 작풍(Style)으로 본문을 재작성.
*   **3D 그래프 시각화:** 에피소드 간의 인물 및 사건 관계도를 3D 그래프(Three.js)로 시각화하여 스토리 구조 파악 지원.
*   **파일 내보내기 (Export):** 완성된 소설을 TXT, EPUB, PDF 포맷으로 변환하여 다운로드.

## 3. 기술 스택
*   **Frontend:** React 19, TypeScript, Vite, React Router, Three.js (`react-force-graph-3d`)
*   **Backend:** Python 3.12, FastAPI, SQLAlchemy (Async), Alembic
*   **Database:** PostgreSQL 16 (with `pgvector`)
*   **AI & LLM:** OpenAI API / Gemini API (텍스트 생성), Text Embedding API (bge-m3 등)
*   **Infrastructure:** Docker, Docker Compose, Nginx

## 4. 담당 역할 및 주요 기술적 의사결정
> **💡 작성 가이드:** 본인의 실제 역할과 기여도에 맞게 수정하여 사용하세요. (예: 풀스택 개발, 백엔드/AI 파이프라인 전담 등)

*   **[담당 역할] 풀스택 개발 및 AI 파이프라인 설계 (100%)**
*   **[기술적 의사결정 1] RAG 아키텍처를 위한 DB 통합 (PostgreSQL + pgvector)**
    *   *배경:* 별도의 벡터 DB(Pinecone, Milvus 등)를 도입하면 시스템 복잡도와 관리 비용이 증가함.
    *   *결정:* 스토리 메타데이터(RDB 구조)와 텍스트 임베딩(Vector) 데이터를 일관성 있게 조인하고 관리하기 위해 `pgvector` 확장을 사용한 단일 PostgreSQL 데이터베이스 채택.
*   **[기술적 의사결정 2] 장편 소설을 위한 계층적 컨텍스트(Hierarchical Context) 관리 기법 도입**
    *   *배경:* LLM의 컨텍스트 윈도우 한계로 인해 수백 페이지의 이전 내용을 프롬프트에 모두 담을 수 없음.
    *   *결정:* 과거 전체 텍스트 대신 **'설정집(Bible) + 전체 스토리 요약 + 최근 N개 에피소드 원문(Sliding Window)'**을 조합하여 컨텍스트를 동적으로 구성하는 파이프라인 구현.
*   **[기술적 의사결정 3] 비동기 처리(Async) 최적화**
    *   *배경:* LLM API 호출 및 벡터 쿼리는 I/O 바운드 작업으로 대기 시간이 긺.
    *   *결정:* FastAPI와 `asyncpg` 기반의 비동기 SQLAlchemy를 사용하여 다중 사용자의 AI 요청을 블로킹 없이 효율적으로 처리.
*   **[기술적 의사결정 4] 일관된 배포 환경을 위한 컨테이너화**
    *   *결정:* 프론트엔드(Nginx 프록시), 백엔드, DB를 묶는 `docker-compose` 환경을 구축하여 로컬 개발과 프로덕션 배포의 환경 차이를 없애고 단일 명령어로 즉시 실행 가능한 환경 제공.

## 5. 프로젝트 주요 성과
> **💡 작성 가이드:** 구체적인 수치나 정성적 성과를 본인 경험에 맞게 채워주세요.

*   **RAG 파이프라인 자체 구축:** 서드파티 프레임워크(LangChain 등)에 의존하지 않고 커스텀 컨텍스트 빌더와 청크 분할 및 벡터 검색 로직을 직접 구현하여 시스템 이해도 증명 및 최적화 달성.
*   **자동화된 설정 관리:** 사용자가 수동으로 설정집을 관리하던 시간을 절약하고, AI가 자동으로 인물/장소를 추출(JSON Array 파싱)하여 데이터베이스에 반영하는 자동화 플로우 구축.
*   **안정적인 아키텍처 설계:** 백엔드/프론트엔드/DB의 명확한 역할 분리와 REST API 설계를 통해 확장성 있는 서비스 구조 완성.
*   **풀스택 기술 역량 증명:** Docker를 활용한 인프라 구성부터 DB 마이그레이션(Alembic), 비동기 API 서버 구현, React 기반의 인터랙티브 UI(3D 그래프 등) 개발까지 전 과정을 주도적으로 수행.
